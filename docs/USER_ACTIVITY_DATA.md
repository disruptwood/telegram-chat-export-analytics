# User Activity Data

This document describes the user-activity datasets generated by the project, where they live, and how to interpret them.

## Source

- All datasets are built from Telegram `result.json` exports.
- User-activity analytics use only messages of type `message`.
- By default, participant analytics include only senders whose `from_id` starts with `user`.
- `channel...` and other non-user senders are preserved in `sender_directory.csv`, but excluded from the main user-activity series by default.
- Important limitation: Telegram export does not expose a reliable `is_bot` flag, so user-like bot accounts may still remain in the data.

## Time Resolutions

User activity is currently exported at four resolutions:

- `day`
- `week`
- `month`
- `year`

For each resolution the toolkit produces:

- a per-period summary table
- a top-N sender table
- a JSON structure for animations and sliders
- a global sender directory

## Files

### Per-Period Summary Files

- `outputs/participant_summary_day.csv`
- `outputs/participant_summary_week.csv`
- `outputs/participant_summary_month.csv`
- `outputs/participant_summary_year.csv`

Each row describes one time bucket.

Fields:

- `period`: bucket type (`day`, `week`, `month`, `year`)
- `label`: human-readable period key such as `2026-03-05`, `2026-W10`, `2026-03`, `2026`
- `start_date`: period start
- `end_date`: period end
- `total_messages`: number of included messages sent in the period
- `active_senders`: number of distinct senders with at least one message in the period
- `first_time_senders`: number of senders whose first-ever message in the export falls in the period
- `first_time_sender_share`: share of active senders who are first-time senders
- `core_threshold`: threshold used to classify a sender as part of the active core
- `core_senders`: number of senders who reached `core_threshold` in the period
- `new_core_senders`: number of senders who reached the core threshold for the first time ever in the export
- `retained_core_senders`: number of senders who were core in both the previous period and the current one
- `reactivated_core_senders`: number of senders who had been core before, dropped out, and then became core again
- `top3_share`: share of messages written by the top 3 senders in the period
- `top10_share`: share of messages written by the top 10 senders in the period
- `messages_per_active_sender`: average messages per active sender
- `unique_senders_per_100_messages`: number of distinct active senders per 100 messages
- `effective_senders`: effective number of senders after accounting for concentration
- `breadth_ratio`: `effective_senders / active_senders`; closer to `1` means broader participation, closer to `0` means a conversation dominated by a smaller core
- `concentration_hhi`: concentration index; higher means a larger share of messages comes from a narrow group

### Top-N Sender Files

- `outputs/top_senders_day.csv`
- `outputs/top_senders_week.csv`
- `outputs/top_senders_month.csv`
- `outputs/top_senders_year.csv`

These are long-form tables. Each period contributes up to `N` rows, one per top sender in that bucket.

Fields:

- `period`
- `label`
- `start_date`
- `end_date`
- `rank`: sender rank within the period
- `sender_id`: stable sender identifier
- `display_name`: display name observed for that sender
- `sender_kind`: sender type, usually `user`; may also be `channel`, `chat`, or `other` when non-user senders are included
- `message_count`: number of messages written by the sender in the period
- `share_of_messages`: share of the period's messages written by that sender
- `is_first_time_sender`: whether the sender first appears in the export in this period
- `is_core_sender`: whether the sender reaches the period's active-core threshold

### JSON Files for Animations

- `outputs/top_senders_day.json`
- `outputs/top_senders_week.json`
- `outputs/top_senders_month.json`
- `outputs/top_senders_year.json`

These files are designed for frontends, visual timelines, and slider-based animations.

Each JSON element contains:

- `period`
- `label`
- `start_date`
- `end_date`
- `total_messages`
- `active_senders`
- `core_threshold`
- `core_senders`
- `first_time_senders`
- `top_senders`: array of sender objects with `rank`, `sender_id`, `display_name`, `sender_kind`, `message_count`, `share_of_messages`, `is_first_time_sender`, `is_core_sender`

Practical use:

- CSV is convenient for pandas, SQL, spreadsheets, and manual filtering
- JSON is convenient for animations, browser visualizations, and notebooks with sliders

### Sender Directory

- `outputs/sender_directory.csv`

This is not a time series. It is an inventory of all senders found in the export.

Fields:

- `sender_id`
- `display_name`
- `sender_kind`
- `is_human_like`
- `total_messages`
- `first_message_at`
- `last_message_at`

Use cases:

- inspect who is treated as a participant
- separate channels and other non-user sources
- build manual exclusion lists if later cleaning is needed

## Active-Core Thresholds

Current defaults:

- `day = 1`
- `week = 3`
- `month = 5`
- `year = 20`

These thresholds are practical defaults for identifying a meaningfully active sender. They are not universal truths and can be changed later if the analysis requires a stricter or softer definition.

## How to Read the Metrics

### Broad Conversation Across Many People

Typical signs:

- high `active_senders`
- high `first_time_senders` or `new_core_senders`
- low `top10_share`
- high `breadth_ratio`
- high `unique_senders_per_100_messages`

### Deep Conversation Driven by a Smaller Core

Typical signs:

- high `messages_per_active_sender`
- high `top3_share` and `top10_share`
- low `breadth_ratio`
- low `unique_senders_per_100_messages`

## What You Can Already Do

- track how the active user base changes by day, week, month, and year
- see when new participants arrive and when they become part of the active core
- compare broader periods against more concentrated periods
- build top-10 sender animations directly from the JSON outputs
- compare message spikes with active-sender spikes to distinguish wide participation from denser activity by a smaller core

## Reaction Analytics

### Reaction Profiles

- `outputs/reaction_profiles.csv`

Per-sender reaction statistics for the top N senders by message count.

Fields:

- `sender_id`: stable sender identifier
- `display_name`: display name
- `total_messages`: number of messages by this sender
- `total_chars`: total characters written
- `total_reactions`: sum of all reaction counts across all messages
- `reactions_per_message`: `total_reactions / total_messages`
- `reactions_per_1k_chars`: `total_reactions / (total_chars / 1000)`
- `median_reactions`: median number of reactions per message
- `messages_with_reactions`: number of messages that received at least one reaction
- `messages_with_reactions_share`: share of messages that received at least one reaction

### Reaction by Period

- `outputs/reaction_by_day.csv`
- `outputs/reaction_by_week.csv`
- `outputs/reaction_by_month.csv`
- `outputs/reaction_by_year.csv`

Per-sender per-period breakdown, useful for tracking how reaction rates evolve over time.

Fields:

- `period`, `label`, `start_date`, `end_date`: time bucket
- `sender_id`, `display_name`: sender info
- `messages`: messages sent in the period
- `total_reactions`: sum of reactions in the period
- `reactions_per_message`: period-level R/Msg
- `reactions_per_1k_chars`: period-level R/1kChars
- `messages_with_reactions`: messages with at least one reaction

### Reaction Stability

- `outputs/reaction_stability_month.csv` (period depends on `--stability-period`)

Measures how consistent a sender's reaction rate is across time periods.

Fields:

- `sender_id`, `display_name`: sender info
- `periods_active`: number of periods in which the sender posted
- `periods_with_reactions`: number of periods where messages received reactions
- `mean_reactions_per_message`: average R/Msg across periods
- `std_reactions_per_message`: standard deviation of R/Msg
- `cv_reactions_per_message`: coefficient of variation (std / mean); lower is more stable
- `min_reactions_per_message`, `max_reactions_per_message`: range
- `median_reactions_per_message`: median R/Msg across periods
- `consistency_score`: `mean / (1 + CV)`; rewards high reaction rates with low variance

### Top Reacted Messages

- `outputs/top_reacted_messages.csv`

Messages ranked by total reaction count.

Fields:

- `message_id`: Telegram message ID
- `date`: timestamp
- `sender_id`, `display_name`: message author
- `text_preview`: first 200 characters of the message
- `total_reactions`: total reaction count
- `reaction_breakdown`: space-separated `emoji:count` pairs

### Sender Filtering

Reaction analytics support several strategies for excluding low-activity senders:

- **Top N** (`--top-n-senders`): only include the N senders with the most messages (default)
- **Percentile** (`--min-percentile`): only include senders above a given percentile of the message count distribution
- **Min messages** (programmatic API): `filter_senders_min_messages(messages, min_count)`

The message count distribution statistics are shown in the Markdown report to help choose appropriate thresholds.

### How to Read Reaction Metrics

#### Consistently Popular Sender

Typical signs:

- high `reactions_per_message`
- high `median_reactions` (not just a few viral messages)
- high `messages_with_reactions_share`
- low `cv_reactions_per_message` in stability analysis
- high `consistency_score`

#### Spike Poster (Occasional Viral Messages)

Typical signs:

- moderate or low `median_reactions` despite high `total_reactions`
- high `cv_reactions_per_message`
- low `consistency_score`
- a few messages in `top_reacted_messages.csv` with very high counts

## Limitations

- The datasets measure written messages, not read activity.
- If a sender changes their display name, the exported datasets keep the last observed name for that `sender_id`.
- User accounts without names fall back to their `sender_id`.
- Telegram export does not provide a reliable way to tell whether a `user...` account is actually a bot.
- All metrics operate only within the exported time range; activity before the export start is invisible.
